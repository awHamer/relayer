## Why Relayer?

Over the years, across many projects, the same pattern kept appearing: a repository layer that brings database queries closer to API filters and makes dynamic fields (computed, derived) a first-class part of the data model -- with full support for filtering, sorting, and aggregation. Not as an afterthought, not through raw SQL escape hatches, but as a core design principle.

Relayer is that pattern extracted into a library. Built with **API integration in mind** from day one.

## Key Features

<CardGrid>
  <Card title="Query DSL" icon="magnifier">
    Prisma-like `findMany`, `where`, `select`, `orderBy` with 20+ operators. The DSL is a plain
    JSON-serializable object, making it trivial to wire up as REST/GraphQL filters.
  </Card>
  <Card title="Computed Fields" icon="puzzle">
    Virtual SQL expressions evaluated at SELECT time. No raw queries, no post-processing -- just
    declare and use.
  </Card>
  <Card title="Derived Fields" icon="rocket">
    Automatic subquery JOINs with full filtering and sorting support. Scalar and object-type values
    with type-safe dot notation.
  </Card>
  <Card title="JSON Filtering" icon="list-format">
    Transparent nested path queries on JSON columns with auto type casting for numeric and boolean
    comparisons.
  </Card>
  <Card title="Relation Filters" icon="random">
    `$exists`, `$some`, `$every`, `$none` -- filter parent records based on related data via EXISTS
    subqueries.
  </Card>
  <Card title="Aggregations" icon="bars">
    `_count`, `_sum`, `_avg`, `_min`, `_max` with `groupBy` and dot-notation for cross-table
    grouping via automatic LEFT JOIN.
  </Card>
  <Card title="Typed Context" icon="setting">
    Pass per-request data (current user, tenant, locale) to field resolvers. Fully typed, available
    in computed and derived fields.
  </Card>
  <Card title="Multi-Dialect" icon="translate">
    PostgreSQL, MySQL, and SQLite. Dialect-specific SQL is handled automatically -- `ILIKE`, array
    operators, JSON paths, `RETURNING`.
  </Card>
</CardGrid>

## Quick Example

```ts
import { relations } from 'drizzle-orm';
import { integer, jsonb, pgTable, serial, text } from 'drizzle-orm/pg-core';
import { createRelayerDrizzle, FieldType } from '@relayerjs/drizzle';

// ─── Schema ───────────────────────────────────────────
const users = pgTable('users', {
  id: serial('id').primaryKey(),
  name: text('name').notNull(),
  metadata: jsonb('metadata').$type<{ role: string; level: number }>(),
});

const posts = pgTable('posts', {
  id: serial('id').primaryKey(),
  title: text('title').notNull(),
  authorId: integer('author_id')
    .notNull()
    .references(() => users.id),
});

const usersRelations = relations(users, ({ many }) => ({ posts: many(posts) }));
const postsRelations = relations(posts, ({ one }) => ({
  author: one(users, { fields: [posts.authorId], references: [users.id] }),
}));

// ─── Relayer Client ───────────────────────────────────
const r = createRelayerDrizzle({
  db,
  schema: { users, posts, usersRelations, postsRelations },
  entities: {
    users: {
      fields: {
        // Computed: virtual SQL expression, evaluated at SELECT time
        displayName: {
          type: FieldType.Computed,
          valueType: 'string',
          resolve: ({ table, sql }) => sql`upper(${table.name})`,
        },
        // Derived: automatic subquery JOIN — counts posts per user
        stats: {
          type: FieldType.Derived,
          valueType: { postsCount: 'number', latestTitle: 'string' },
          query: ({ db, schema: s, sql, field }) =>
            db
              .select({
                [field('postsCount')]: sql<number>`count(*)::int`,
                [field('latestTitle')]: sql<string>`max(${s.posts.title})`,
                authorId: s.posts.authorId,
              })
              .from(s.posts)
              .groupBy(s.posts.authorId),
          on: ({ parent, derived, eq }) => eq(parent.id, derived.authorId),
        },
      },
    },
  },
});

// ─── Query — everything is fully type-safe ────────────
const result = await r.users.findMany({
  select: {
    id: true,
    displayName: true, // computed field
    stats: { postsCount: true }, // derived sub-field select
    posts: { id: true, title: true }, // relation loading (no N+1!)
  },
  where: {
    metadata: { role: 'admin' }, // JSON path filtering
    posts: { $some: { title: { contains: 'Relayer' } } }, // relation filter
  },
  orderBy: [
    { field: 'metadata.level', order: 'desc' }, // type-safe JSON path ordering 🎉
    { field: 'stats.postsCount', order: 'desc' }, // type-safe derived field ordering 🎉
  ],
  limit: 10,
});

// ─── Aggregation ──────────────────────────────────────
const stats = await r.posts.aggregate({
  groupBy: ['author.name'], // dot-notation: LEFT JOIN + group by related column
  _count: true,
  _max: { title: true },
});
```

## Inspiration

Relayer is inspired by Prisma query API, Hasura GraphQL filters, nestjs-query, and many other tools that make database access feel effortless.
